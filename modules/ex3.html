Exercice 3 : Tâches Sporadiques & Saturation

Soit le système S2 de trois tâches indépendantes, sporadiques, à échéances sur requête, dont les paramètres $\langle C_i, T_i \rangle$ sont donnés par :

  • $\tau_1 = \langle 2, 7 \rangle$
  • $\tau_2 = \langle 2, 4 \rangle$
  • $\tau_3 = \langle 3, 14 \rangle$

Le système est ordonnancé suivant RM (Rate Monotonic).

Analyse Préalable & Priorités

Sporadiques : $T_i$ représente l'intervalle minimum entre deux activations. Dans le pire cas, elles se comportent comme des périodiques simultanées.

Ordonnancement RM : Priorité par période croissante.

  • $T_2 (4) < T_1 (7) < T_3 (14)$
  • Priorités : $P(\tau_2) > P(\tau_1) > P(\tau_3)$

Attention : L'indice ne correspond pas à l'ordre !

  • $hp(2) = \emptyset$
  • $hp(1) = \{\tau_2\}$
  • $hp(3) = \{\tau_2, \tau_1\}$

a) Calculer la charge du système S2, et conclure.

$$ U = \frac{2}{7} + \frac{2}{4} + \frac{3}{14} = \frac{4}{14} + \frac{7}{14} + \frac{3}{14} = \frac{14}{14} = \mathbf{1} $$

Conclusion : La charge est de 100%. Le processeur sera utilisé sans temps mort. Système critique, RTA indispensable.

b & c) Optimalité RM et Précision RTA ?

Optimalité : Oui, RM reste optimal. Les tâches sporadiques ont $D_i = T_i$.

Précision : Exacte. Le scénario d'avalanche (toutes activées à $t=0$ puis systématiquement au plus tôt) est physiquement possible pour des tâches sporadiques.

d) Calculer avec la RTA le temps de réponse (Lehoczky).

1. Tâche $\tau_2$ (Priorité 1)

Point fixe atteint immédiatement : $w_2 = 2$. Ordonnançable ($2 \le 4$).

2. Tâche $\tau_1$ (Priorité 2)

  • $w_1^0 = C_1 = 2$
  • $w_1^1 = 2 + \left\lceil \frac{2}{4} \right\rceil \times 2 = 2 + 2 = 4$
  • $w_1^2 = 2 + \left\lceil \frac{4}{4} \right\rceil \times 2 = 2 + 2 = 4$

Point fixe : $w_1 = 4$. Ordonnançable ($4 \le 7$).

3. Tâche $\tau_3$ (Priorité 3) - L'avalanche

Calcul pour la 1ère instance ($k=1$) :

  • $w_3^0(1) = 3$
  • $w_3^1(1) = 3 + \left\lceil \frac{3}{4} \right\rceil \times 2 + \left\lceil \frac{3}{7} \right\rceil \times 2 = 3 + 2 + 2 = 7$
  • $w_3^2(1) = 3 + \left\lceil \frac{7}{4} \right\rceil \times 2 + \left\lceil \frac{7}{7} \right\rceil \times 2 = 3 + 4 + 2 = 9$
  • $w_3^3(1) = 3 + \left\lceil \frac{9}{4} \right\rceil \times 2 + \left\lceil \frac{9}{7} \right\rceil \times 2 = 3 + 6 + 4 = 13$
  • $w_3^4(1) = 3 + \left\lceil \frac{13}{4} \right\rceil \times 2 + \left\lceil \frac{13}{7} \right\rceil \times 2 = 3 + 8 + 4 = 15$
  • $w_3^5(1) = 3 + \left\lceil \frac{15}{4} \right\rceil \times 2 + \left\lceil \frac{15}{7} \right\rceil \times 2 = 3 + 8 + 6 = 17$
  • $w_3^6(1) = 3 + \left\lceil \frac{17}{4} \right\rceil \times 2 + \left\lceil \frac{17}{7} \right\rceil \times 2 = 3 + 10 + 6 = 19$
  • $w_3^7(1) = 3 + \left\lceil \frac{19}{4} \right\rceil \times 2 + \left\lceil \frac{19}{7} \right\rceil \times 2 = \mathbf{19}$

Formule de Lehoczky (Débordement)

Le point fixe de la 1ère instance est 19. Or, $19 > T_3 (14)$. La 2ème instance arrive avant que la 1ère ne finisse ! La "période d'activité" continue.

Calcul pour la 2ème instance ($k=2$, demande $2 \times C_3 = 6$) :

  • $w_3^0(2) = 19$ (On repart du dernier point)
  • $w_3^1(2) = 6 + \left\lceil \frac{19}{4} \right\rceil \times 2 + \left\lceil \frac{19}{7} \right\rceil \times 2 = 6 + 10 + 6 = 22$
  • $w_3^2(2) = 6 + \left\lceil \frac{22}{4} \right\rceil \times 2 + \left\lceil \frac{22}{7} \right\rceil \times 2 = 6 + 12 + 8 = 26$
  • $w_3^3(2) = 6 + \left\lceil \frac{26}{4} \right\rceil \times 2 + \left\lceil \frac{26}{7} \right\rceil \times 2 = 6 + 14 + 8 = 28$
  • $w_3^4(2) = 6 + \left\lceil \frac{28}{4} \right\rceil \times 2 + \left\lceil \frac{28}{7} \right\rceil \times 2 = \mathbf{28}$

Fin de l'activité car $28 \le 2 \times T_3$ ($28 \le 28$).

Le temps de réponse de cette 2ème instance est : $28 - (1 \times 14) = \mathbf{14}$.

Le pire temps de réponse final est le max : $\max(19, 14) = \mathbf{19}$.

Conclusion Finale

Le pire temps de réponse de $\tau_3$ est 19. Puisque $19 > D_3 (14)$, le système S2 N'EST PAS ordonnançable sous RM. C'est l'exemple type d'une saturation due aux interférences malgré une charge CPU valide ($U=1$).
